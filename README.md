# FinTech Fraud Detection Pipeline (Lambda/Kappa Style)

Production-style academic project implementing a real-time and batch-capable fraud detection platform for a digital wallet provider.

## Architecture Overview

Flow:

Producer -> Kafka -> Spark Structured Streaming -> PostgreSQL -> Airflow -> Reports

- Real-time path (speed layer):
	- Python producer generates synthetic transactions and controlled fraud events.
	- Kafka transports transaction events using the `transactions` topic.
	- Spark Structured Streaming parses JSON, applies fraud rules in event time, and writes immediate fraud alerts to PostgreSQL.
- Batch path (batch layer):
	- Airflow runs every 6 hours.
	- Airflow reads validated transactions and fraud alerts from PostgreSQL.
	- It produces reconciliation and analytics CSV reports in `/reports`.

## Fraud Rules Implemented

1. High-value fraud:
	 - `amount > 5000`
2. Impossible travel fraud:
	 - Same `user_id` transacts from different countries within 10 minutes (event-time based).

## Event Time vs Processing Time (Assignment Note)

- Event time:
	- Taken from each transaction payload field `timestamp`.
	- Represents when the transaction occurred in business reality.
- Processing time:
	- When Spark receives and processes the record.
	- Can differ due to network delays, retries, and out-of-order ingestion.

Why this matters for impossible travel:
- Fraud logic must compare transaction occurrence times, not ingestion times.
- The Spark job parses `timestamp` into `event_ts`, applies `withWatermark(...)`, and uses stateful per-user tracking (`applyInPandasWithState`, equivalent to `mapGroupsWithState`) to compare the current location/time with the previous user event.

## Build Only (No Auto Execution)

Build all images:

```bash
docker compose build
```

Bring up infrastructure:

```bash
docker compose up -d kafka postgres airflow-init airflow-webserver airflow-scheduler producer spark
```

Create Kafka topics manually:

```bash
docker compose run --rm kafka-topic-init
```

## Manual Runtime Steps

### 1. Run Producer Manually

The producer service is intentionally kept idle (`sleep infinity`).

```bash
docker compose exec producer python /app/producer/main.py
```

### 2. Run Spark Streaming Job Manually

Submit the streaming job explicitly from the spark container:

```bash
docker compose exec spark spark-submit \
	--packages org.apache.spark:spark-sql-kafka-0-10_2.12:3.5.1,org.postgresql:postgresql:42.7.3 \
	/app/spark/fraud_detection.py
```

### 3. Trigger Airflow DAG Manually

The DAG is named `fraud_reconciliation_dag` and is scheduled every 6 hours (`0 */6 * * *`).

Manual trigger command:

```bash
docker compose exec airflow-webserver airflow dags trigger fraud_reconciliation_dag
```

Optional check:

```bash
docker compose exec airflow-webserver airflow dags list
```

## Data Model Summary

Input transaction schema:

```json
{
	"user_id": "U1001",
	"timestamp": "2026-05-10T10:00:00+00:00",
	"merchant_category": "electronics",
	"amount": 123.45,
	"location": "Sri Lanka"
}
```

PostgreSQL tables:

- `fraud_alerts`:
	- `id, user_id, timestamp, amount, merchant_category, location, fraud_reason, created_at`
- `validated_transactions`:
	- `id, user_id, timestamp, amount, merchant_category, location, created_at`
